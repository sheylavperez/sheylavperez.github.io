Introduced a redesigned Topic Page that 65% of participants preferred, featuring a teacher-centered layout with clear standards and activity guidance.

Project Overview

An independent poll of district admins gave BrainPOP an average NPS score of -36. When districts like Baltimore decide to not renew BrainPOP, they cite reasons such as a “lack of instructional rigor” and “misalignment with classroom best practices.” We need to make the instructional intention and value more apparent in key areas of the BrainPOP experience. The topic page is an integral part of the teacher and admin flow, so it presents a good opportunity to:

  1. Indicate how BrainPOP supports classroom instruction
  2. Make it easy for teachers to incorporate BrainPOP in the classroom

I’ll be implementing changes to the topic page in two phases, but will use this study to validate changes beyond the first iteration so we’re set up for the future.

The Solution

  • ✓
    Redesigned topic page focused on instructional rigor and teacher needs
  • ✓

The Process

Research
I conducted user testing and feedback analysis to understand user needs and challenges with the current Topic Page.
  • User testing with 17 participants (teachers and admins)
  • Click and interaction heatmaps on Topic Page elements
  • Feedback on instructional rigor and planning support
Strategy
Using research insights, I developed a phased redesign strategy focused on improving instructional clarity and teacher usability.
  • Resarch Plan
  • Prioritize standards and instructional metadata
  • Create teacher-centered layout with guides and activity sets
  • Plan onboarding to ease transition for existing users
Design
Designed and iterated prototypes emphasizing clarity, ease of planning, and better decision-making for teachers and admins..
  • Teacher Guide and activity metadata front and center
  • Activity cards grouped by classroom routines (e.g., Class Discussion, Reading Focus)
  • Simplified layout reducing cognitive load
Implementation
Roll out changes in two phases while continuously gathering user feedback to refine features and onboarding.
  • Phase 1: Launch new Topic Page with teacher guides and metadata
  • Phase 2: Introduce Activity Sets and concise teacher advice
  • Provide early notification and onboarding resources
Evaluation & Iteration
Monitor usage and user feedback to refine features, optimize filters, and improve user experience over time.
  • Track clicks and engagement with standards and activity sets
  • Conduct follow-up research to finalize filters and simplify copy
  • Support users to adapt with clear guidance and training

Result

Project Outcomes

  • ★
    43% Increase in Sales – Admins’ improved perception of instructional rigor and alignment influenced purchasing and renewal decisions.
  • ★
    65% Clicked “Standards Covered” – Participants actively engaged with the standards to assess instructional fit.
  • ★
    2x More Admin Activity – Admins interacted with Activity Sets twice as often as teachers, showing higher engagement with key planning tools.
  • ★
    4.8 / 5 Average Rating – Participants rated the redesigned Topic Page highly for intuitiveness and support in instruction.

Lessons Learned

  • ✦
    Early user testing is crucial. Small feedback loops save redesign time later.
  • ✦
    Clear navigation improves both engagement and efficiency.
  • ✦
    Collaboration between designers and educators helps align design with classroom needs.
  • ✦
    Data from usability testing should always guide final design decisions.
