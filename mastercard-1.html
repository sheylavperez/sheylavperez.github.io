Project Overview

Banks often face challenges when assessing potential merchants for partnerships. An internal survey revealed that bankers struggled to quickly evaluate merchant credibility, leading to delayed decisions and increased risk. The platform needed to provide clear, actionable insights on merchant backgrounds, crypto activity, portfolio history, and fraud reports.


The redesigned dashboard aimed to:

  1. Give banks a clear, consolidated view of merchant backgrounds
  2. Reduce time spent analyzing multiple data sources
  3. Increase confidence in partnership decisions

The Process

Research
Interviewed 12 banking users to understand pain points, map workflows, and assess trust in merchant data.
  • Collected feedback on dashboard usability and clarity
  • Mapped workflows for merchant evaluation
  • Identified key data points for decision-making
Strategy
Used research insights to prioritize data points and design a dashboard layout highlighting high-risk flags and actionable insights.
  • Prioritize merchant background, crypto activity, portfolio, and fraud reports
  • Plan onboarding and training materials for bank users
  • Define success metrics for trust and decision-making speed
Design
Created wireframes and high-fidelity prototypes in Figma, focusing on clarity, hierarchy, and actionable insights for banking decisions.
  • Organized information: Background, Crypto, PortfolioPost, Fraud Reports
  • Interactive dashboard elements for faster evaluation
  • Simplified layout reducing cognitive load for bank users
Implementation
Built 2 interactive dashboards, iterated with feedback from 5+ stakeholders, and documented workflows to streamline onboarding.
  • Developed dashboards for merchant insights and risk flags
  • Incorporated stakeholder feedback into iterative design
  • Maintained 12+ Jira documents for guides, workflows, and training
Evaluation & Iteration
Monitored usage and feedback to optimize dashboard features, improve decision-making speed, and increase trust in merchant evaluations.
  • Tracked dashboard engagement and decision-making efficiency
  • Refined features based on follow-up interviews
  • Updated onboarding and guidance for new bank users

Project Gallery

Onboard Risk Check

Design: The dashboard's design prioritizes a clean, grid-based layout to present complex merchant data clearly, allowing for rapid scanning and comparison. A minimalist interface and the use of visual cues help to reduce clutter and cognitive load.

Reasoning: This design strategy is intended to boost bank confidence and significantly speed up decision-making by consolidating fragmented information into a single, cohesive view.

Cyber Security Dashboard

Design: This dashboard uses a card-based layout to organize data, with key metrics highlighted at the top. Visualizations like charts provide quick insights, and tables with color-coding are used for detailed information.

Reasoning: The design makes complex data digestible and actionable by reducing cognitive load. Separating information and using visuals allows users to quickly identify key trends, while tables help prioritize the most critical cases efficiently.

Crypto Risk Evaluation

Design: This dashboard uses a single-column layout with a large, isolated card for the overall risk score. Information is segmented into distinct sections with a clear "Yes/No" format for key data points.

Reasoning: The design is engineered for rapid risk assessment by placing the most critical metric in a prominent card. Simplifying technical information into a simple "Yes/No" format reduces cognitive load, allowing users to quickly grasp risk factors and make efficient decisions

Inquiry Details

Design: This page uses a simple, single-column layout with a left-hand navigation bar. It presents merchant details in a clean, stacked format, using clear labels and bold headings for key data points.

Reasoning: The design provides a clear, detailed view of information without distractions. The navigation bar improves workflow efficiency, while the stacked layout and bold labels ensure that key details are instantly scannable and easy to verify.

Result

Project Outcomes

  • ★
    50% Faster Decision-Making: By consolidating fragmented data onto three new dashboards, the project reduced the time banks needed to assess merchants by 50%.
  • ★
    6% Increase in Bank Confidence: The platform's ability to provide secure, actionable insights on merchant backgrounds directly led to a 5% boost in bank confidence.

Lessons Learned

  • ✦
    Trust is a Product Feature: Gaining user trust, especially in high-stakes industries, is crucial. It's built through transparent data sources and clear visualizations, not just features.
  • ✦
    Simplicity is Strategic: A clean, minimalist design isn't just about aesthetics; it directly impacts key business outcomes by reducing cognitive load and increasing decision speed.
